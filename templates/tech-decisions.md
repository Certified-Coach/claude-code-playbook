# Tech Decisions Log — [PROJECT NAME]

> Record architectural and technology decisions here. Each entry explains
> what was decided, why, and what alternatives were considered.
> This prevents revisiting the same decisions and helps onboard new contributors.

## How to use this

When you make a technology choice (framework, library, service, pattern), add an entry.
Format:

```
### DEC-NNN: [Title]
**Date:** YYYY-MM-DD
**Status:** Decided / Superseded by DEC-NNN / Under review

**Decision:** [One sentence — what we're doing]

**Context:** [Why this decision was needed]

**Alternatives considered:**
- [Option A] — [why not]
- [Option B] — [why not]

**Consequences:** [What this means for the project going forward]
```
